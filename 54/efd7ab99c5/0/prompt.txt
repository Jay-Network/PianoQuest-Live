# PianoQuest Live: Python → TypeScript ADK Migration Plan

## Context

Jay directed immediate migration from Python ADK to TypeScript ADK (`@google/adk`). The existing Python app (v0.4.7) uses FastAPI + WebSocket + Google ADK with 5 tool functions. ADK TS v0.5 does NOT have `Runner.runLive()` (TODO in source). BookSage_Live solved this by using `@google/genai` Live API directly with manual tool declarations — we follow the same pattern.

## Critical Finding

**`Runner.runLive()` is not implemented in ADK TS v0.5** (line 292 of runner.js: `// TODO - b/425992518`). BookSage_Live's working pattern: use `@google/genai`'s `client.live.connect()` directly, declare tools as `functionDeclarations`, execute tools locally, and send `toolResponse` back to Gemini. We adopt this exact pattern.

## Architecture

- **Runtime**: Node.js + Express + `ws` (raw WebSocket)
- **AI**: `@google/genai` Live API (NOT ADK Runner for live streaming)
- **Tools**: Manual `functionDeclarations` + local execution (same pattern as BookSage)
- **ADK**: `LlmAgent` + `FunctionTool` definitions kept for structure/future `runLive` support
- **Frontend**: `static/index.html` — UNCHANGED (same WebSocket protocol)

## WebSocket Protocol (UNCHANGED)

Client → Server:
- Binary: PCM audio at 16kHz
- `{type: "video_frame", data: base64}`: Camera JPEG
- `{type: "text", content: string}`: Text message
- `{type: "midi_snapshot", ...}`: MIDI data
- `{type: "camera_state", enabled: bool}`: Camera toggle
- `{type: "hand_state", detected_count, both_hands_detected}`: Hand tracking
- `{type: "set_mode", mode}`: Agent mode
- `{type: "close"}`: Disconnect

Server → Client:
- Binary: PCM audio at 24kHz (Gemini output)
- `{type: "input_transcript", text}`: User speech
- `{type: "output_transcript", text}`: Agent speech
- `{type: "turn_complete"}`: Turn done
- `{type: "story_scene", scene_id}`: Tool: scene change
- `{type: "achievement", badge_id}`: Tool: badge
- `{type: "coaching_focus", text}`: Tool: coaching tip
- `{type: "quest_advance", phase}`: Tool: quest progress
- `{type: "technique_report", finger, visual, audio, suggestion}`: Tool: technique

## Key Difference from Python Version

The Python version used ADK's `Runner.run_live()` which managed the Gemini Live session internally, yielding ADK `Event` objects. In TypeScript, we use `@google/genai`'s `client.live.connect()` directly, which means:

1. **Tool execution is manual** — we intercept `response.toolCall`, execute locally, send `session.sendToolResponse()`
2. **Transcription handling differs** — access via `response.serverContent.inputTranscription` instead of ADK event fields
3. **Audio is raw** — `response.data` is binary audio, sent directly to client
4. **No ADK event abstraction** — we handle the raw Gemini Live protocol

This is exactly how BookSage_Live works and is proven in production.
