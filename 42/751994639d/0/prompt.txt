# Plan: Camera Finger Tracking (Hybrid: MediaPipe + Gemini Vision)

## Context
PianoQuest Live competition deadline is TODAY (March 16). Jay wants camera finger tracking integrated with the drill system. Camera/video existed in v1.2-1.5 but was stripped in v2.0-2.1. Full restoration guide at `docs/removed-camera-finger-tracking.md`. Prioritizing Gemini features for maximum demo impact.

**Hybrid approach (Option C):**
1. Phone camera sends JPEG frames (1fps) to Gemini for vision coaching
2. MediaPipe HandLandmarker runs client-side on phone for hand detection + skeleton overlay
3. Gemini sees hands + hears piano + gets MIDI = multimodal coaching (the killer demo feature)

### 3b. `static/index.html` — HandLandmarker init

In `startSecondaryCamera()`, after video is playing:
1. Load `FilesetResolver.forVisionTasks()` with WASM CDN path
2. Create `HandLandmarker` with `float16` model, `runningMode: "VIDEO"`, `numHands: 2`
3. Add overlay canvas on top of video element (same dimensions)

### 3c. `static/index.html` — Hand processing loop

`processHands(timestamp)` via requestAnimationFrame:
- Throttle to ~15fps (66ms between calls)
- `handLandmarker.detectForVideo(video, timestamp)`
- Draw skeleton on overlay canvas using HAND_CONNECTIONS
- Track hand count, send `hand_state` only on change
- Color-code: 0 hands = red badge, 1 = yellow, 2 = green

### 3d. Hand skeleton drawing

`drawHandOverlay(landmarks)`:
- Clear overlay canvas
- For each hand: draw HAND_CONNECTIONS as white lines, keypoints as colored circles
- Fingertips (indices 4,8,12,16,20) get larger circles
- Semi-transparent so video shows through
